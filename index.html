2 - Que es Git?


Git es un sistema de control de versiones creado por Linus Torvalds (Creador de Linux) con el fin de llevar un registro de los cambios realizados en un proyecto de desarrollo de software.


Git presenta una arquitectura distribuida (Sistema de control de versiones distribuido DVCS), esto implica que en lugar de guardar el historial en un espacio centralizado, cada copia de cada desarrollador guarda el historial completo de los cambios.

3 - Por que utilizar Git


Git nos permite hacer un seguimiento de los cambios que se han realizado en un proyecto, por quien se han realizado, en que momento, etc. De esta manera facilita ademas del desarrollo de dicho proyecto, la coordinacion con un grupo de trabajo.


Git es un sistema de control de versiones distribuido (DVCS), potente, agil y de software libre. Todas estas caracteristicas hacen de Git un sistema de control de versiones perfecto para llevar a cabo nuestros proyectos

4 - Listado de comandos basicos en Git


  • git init

    Inicializa un repositorio.


  • git add <nombre_archivo>

    Comienza a trackear el archivo indicado


  • git status

    Muestra el estado de la rama, con los cambios que hay que commitear


  • git commit -m "message"

    Realiza un commit y lo almacena con el mensaje introducido.

    Confirma los cambios realizados.


  • git fetch

    Conecta con el repositorio remoto para ver los cambios pero no llega a cambiar nada en el repositorio local.


  • git pull

    Trae cambios de un repositorio remoto.


  • git push

    Sube los cambios realizados a un repositorio remoto.


  • git branch

    Lista las ramas, tambien serviria para crear un rama poniendo el nombre delante


  • git merge <nombre_rama>

    Fusiona la rama actual con la rama especificada.

5 - Creacion de un proyecto con Git


Para comenzar crearemos el directorio donde ira nuestro proyecto. Desde git bash ejecutamos el siguiente comando:


mkdir "ProyectoGitDespliegue"

Crearemos un archivo README donde escribiremos la informacion importante relativa a nuestro proyecto:


echo "# Proyecto Despliegue" >> README.md

Ahora inicializaremos el repositorio y añadiremos nuestro archivo README:


git init
git add README.md

A continuacion realizaremos nuestro primer commit:


git commit -m "first commit"

Como nuestra intencion es almacenar nuestro repositorio de manera remota, deberemos elegir que plataforma deseamos usar para subir nuestro codigo, en este caso utilizaremos GitHub, pero podriamos usar otras alternativas como GitLab, BitBucket, etc.


Para iniciar el repositorio remoto deberemos entrar a GitHub, registrarnos y seleccionar la opcion que nos permite crear un nuevo repositorio. GitHub nos proporcionara un enlace al repositorio para que podamos vincularlo con nuestro repositorio local.


Ahora con el comando git remote vincularemos nuestro repositorio remoto con el local y subimos nuestra rama master (-u trackeara nuestra rama, esto afectara al git pull):


git remote add origin https://github.com/Trejerum/ProyectoDespliegue.git
git push -u origin master

De esta forma ya tendriamos nuestra rama con el primer commit guardada tanto de manera local como remota. Ahora para continuar con el desarrollo crearemos una rama de desarrollador donde poder efectuar cambios sin que afecten a la rama principal (master) y nos moveremos a ella:


git branch dev-diego
git checkout dev-diego

A partir de este punto el desarrollo consistiria en introducir cambios, añadirlos, commitearlos y subirlos a nuestra rama en remoto.


Finalmente cuando tengamos una version probada y queramos pasarla a la rama principal (master) deberemos fusionar la rama principal con la nuestra. Para esto primero nos bajariamos las ultimas versiones de los commits del repositorio remoto, nos moveriamos a la rama master y ejecutariamos un merge:


git fetch master
git checkout master
git pull
git merge dev-diego
