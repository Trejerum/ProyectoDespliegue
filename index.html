2 - Que es Git?


Git es un sistema de control de versiones creado por Linus Torvalds (Creador de Linux) con el fin de llevar un registro de los cambios realizados en un proyecto de desarrollo de software.


Git presenta una arquitectura distribuida (Sistema de control de versiones distribuido DVCS), esto implica que en lugar de guardar el historial en un espacio centralizado, cada copia de cada desarrollador guarda el historial completo de los cambios.

3 - Por que utilizar Git


Git nos permite hacer un seguimiento de los cambios que se han realizado en un proyecto, por quien se han realizado, en que momento, etc. De esta manera facilita ademas del desarrollo de dicho proyecto, la coordinacion con un grupo de trabajo.


Git es un sistema de control de versiones distribuido (DVCS), potente, agil y de software libre. Todas estas caracteristicas hacen de Git un sistema de control de versiones perfecto para llevar a cabo nuestros proyectos

4 - Listado de comandos basicos en Git


  • git init

    Inicializa un repositorio.


  • git add <nombre_archivo>

    Comienza a trackear el archivo indicado


  • git status

    Muestra el estado de la rama, con los cambios que hay que commitear


  • git commit -m "message"

    Realiza un commit y lo almacena con el mensaje introducido.

    Confirma los cambios realizados.


  • git fetch

    Conecta con el repositorio remoto para ver los cambios pero no llega a cambiar nada en el repositorio local.


  • git pull

    Trae cambios de un repositorio remoto.


  • git push

    Sube los cambios realizados a un repositorio remoto.


  • git branch

    Lista las ramas, tambien serviria para crear un rama poniendo el nombre delante


  • git merge <nombre_rama>

    Fusiona la rama actual con la rama especificada.
